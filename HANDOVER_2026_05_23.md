## Critical API Convention

`ask_with_usage()` in `services/anthropic_client.py` takes:
- `system_prompt` (not `system`)
- `user_prompt` (not `user`)
- Returns a **dict** with keys: `text`, `input_tokens`, `output_tokens`, `web_searches`, `error`
- **Never unpack as a tuple.**

`get_active_product()` returns dict with `product_id` (NOT `id`) and `product_name` (NOT `name`).

## Architecture

One app, one database, one UI. Seven modules coordinate via the database:

```
Researcher   → research_items table
Strategist   → story_angles table
Copywriter   → drafts table

Orchestrator → calls all agents, no own table
```

## Known Issues / Watch List

- **Strategist intermittent JSON parse failure:** Occurs occasionally with large research
  libraries (16+ items). Workaround: re-run — succeeds on retry. Fix before Phase 2.
- **`batch_review_remaining.py` implicit retry:** `review_draft()` returns cached review
  if one exists. Safe to re-run but silently re-triggers API on new drafts with prior
  failures. Add `force=False` guard before next batch use.
- **Orphan editor_reviews rows:** Old drafts #1–12 deleted by regeneration but reviews
  remain. Harmless noise. Cleanup later.
- **HINGLISH_UNCALLED monitor:** Draft #19 used "maidan" once. Editor caught it. Monitor.
